# Information Architecture

- **Organization of information**
    - How data is logically organized
- **Labeling (representation) of information**
    - Provides context for the presented information
- **Navigation systems**
    - How users will find their way around the site
- **Search systems**
    - Ways of finding content

---

# Organizational System

- Organizational **scheme**
    - What criteria we use to categorize information
- Organizational **structure**
    - How we organize the individual categories

---

# Organizational Scheme

- **Exact schemes** — we must know exactly what we are looking for
    - Alphabetical
    - Chronological
    - Geographical
- **Ambiguous schemes**
    - Topical — article categories, product types, ...
    - By activities — what the user wants to achieve
    - By visitor roles — e.g. municipal website: resident, tourist, journalist, ...
    - Metaphorical — e.g. organizing the site like a department store (entrance, information desk, departments, ...)
    - ...or a combination of the above

---

# Organizational Structures

- **Hierarchical**
    - Easy to understand and remember
    - Easy navigation
- **Database-like**
    - Equivalent documents categorized by metadata
    - E.g. user profile pages
- **General graph**
    - Hypertext links between documents
    - Difficult to navigate

---

# Labeling Information on the Web

- Headings (labels)
    - Can form a hierarchy
- Contextual links
    - Choice of link text
- Navigation system
    - Labeling items — home page, search, ...
- Icons
    - Improve the visual appearance
    - Only a limited number of distinguishable icons

---

# Important Properties of Labels

- **Specificity**
    - Avoid overly generic labels
- **Consistency**
    - *Style* — punctuation, capitalisation
    - *Presentation* — fonts, colours
    - *Syntax* — naming sections with nouns, questions, etc.
    - *Granularity* — labelling at the same level of abstraction
    - *Completeness* — logically related information in the same place
    - *Terminology* — technical vs. general terms

---

# Types of Navigation

- Main navigation types
    - *Global* — which part of the site we are in
        - Always visible
    - *Local* — what is in this part (e.g. chapter)
        - Varies across sections, should be consistent
    - *In-text links*
        - Hypertext links to further related information
- Aids
    - *Site map*
    - *Index*
    - *Guide* — linear, step-by-step

---

# Creating Context

- The user should always know
    - Which website they are on
    - Which section they are in
- A search engine may send a user "into the middle" of a site
- There should always be a clear path
    - To the home page
    - To the beginning of the current section
    - One level up
    - From the root page to the current page (breadcrumb)

---

# Search

- Not always necessary
- Demanding to implement and operate
    - Or use e.g. [Google Programmable Search Engine](https://programmablesearchengine.google.com/about/)
- Useful for sites that:
    - Have a large amount of content
    - Have frequently changing content
    - Have an unclear standard navigation
    - Where users expect it
- Search can be over
    - Document content (full-text)
    - Metadata (more demanding, more precise)

---

# Search System Features

- Interface
    - Search box
    - Advanced search
- Search specification
    - Boolean operators
    - Criteria (restrict which documents are searched)
    - Result presentation options (sorting, detail level)
